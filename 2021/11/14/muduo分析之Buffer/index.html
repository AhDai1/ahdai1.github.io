Buffer的设计

1、Buffer对外表现为一块连续的内存(char *, len)。
2、其size()可以自动增长。
3、内部以vector来保存数据。

从前一文对TcpConnection的分析来看，TcpConnection会有两个Buffer成员，input buffer和output buffer。

input buffer：TcpConnection会从socket中读取数据，然后写入到input buffer中，客户代码从input buffer中读取数据。
output buffer：客户代码会将数据写入output buffer中，TcpConnection从output buffer中读取数据并写入socket。

Buffer的数据结构

Buffer的内部是一个vector，是一块连续的内存。数据结构图如下：

在Buffer中有三个部分，prependable、readable、writable。各块的大小为：
prependable = readIndex

readable = writeIndex - readIndex

writable = size() - writeIndex

readFd()

上一文讲到的TcpConnection读取数据的时候，用到了readFd()函数，在这节进行简要介绍：
具体做法是，在栈上准备一个 65536 字节的 stackbuf，然后利用 readv() 来读取数据，iovec 有两块，第一块指向Buffer 中的 writable 字节，另一块指向栈上的 stackbuf。这样如果读入的数据不多，那么全部都读到 Buffer 中去了；如果长度超过 Buffer 的 writable 字节数，就会读到栈上的 stackbuf 里，然后程序再把 stackbuf 里的数据 append 到 Buffer 中。

iovec

一个与readv()和writev()操作相关的结构体：

readv和writev函数用于在一次函数调用中读、写多个非连续缓冲区。这两个函数称为散布和聚集写。
iov_base：指向一个缓冲区，这个缓冲区是存放readv所接收的数据或是writev将要发送的数据。
iov_len：确定了接收的最大长度以及实际写入的长度。
